# Contributing

Thanks for contributing to `VIBE-Claude-Plugin`.

This project is trying to make Claude Code workflows more structured, more repeatable, and easier to operate on larger tasks. Good contributions improve clarity, execution quality, or onboarding.

## Before You Start

Please read:

- `README.md`
- `AI_DEVELOPMENT_DOC.md`
- `LAUNCH_PLAN.md` if your change affects public positioning

## What Makes a Good Contribution

Strong contributions usually improve one of these areas:

- workflow reliability
- installation and onboarding
- documentation clarity
- testing and review flow
- better examples and case studies

## Contribution Rules

Keep changes focused.

- fix one problem per pull request when possible
- avoid unrelated refactors
- preserve the distinction between `/vibe` and `vibe-autopilot`
- keep public-facing writing in English unless the file is intentionally Chinese
- update docs when behavior or public messaging changes

## Suggested Workflow

1. fork the repository
2. create a branch for your change
3. make the smallest useful edit
4. update docs if needed
5. open a pull request with a clear summary

## Pull Request Checklist

Before opening a PR, confirm:

- the change is scoped and understandable
- any user-facing behavior is documented
- naming is consistent across README, docs, and templates
- the PR explains the problem and the fix

## Good First Contributions

If you want a practical starting point, consider:

- improving install instructions
- adding screenshots or demo assets
- polishing launch documents
- tightening one workflow description
- adding a reproducible case study

## Questions

If you are unsure whether a change fits the project, open an issue first and describe:

- the problem you see
- why it matters
- the smallest fix you propose
