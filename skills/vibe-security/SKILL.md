# 安全审查工作流

## 1. 文档目的
在代码合入或交付前，对项目进行严格的安全审查，发现并修复潜在漏洞。

## 2. 核心审查点
- **注入漏洞**：SQL注入、XSS跨站脚本、命令注入防御检查。
- **认证与授权**：越权访问检查、Token/Cookie安全管理。
- **数据安全**：敏感信息（密码、密钥）是否硬编码，是否进行了合理的加密/脱敏。
- **依赖安全**：第三方库是否存在已知漏洞（CVE）。

## 3. 详细流程
### 步骤一：静态代码分析
1. 扫描核心业务逻辑、路由层和数据库操作层。
2. 检查所有的用户输入处理逻辑，确保经过了严格的过滤和转义。

### 步骤二：漏洞模拟与修复
1. 针对发现的安全隐患，提供具体的修复方案。
2. 与用户沟通并执行代码修复，确保安全性与业务功能的平衡。

### 步骤三：输出安全审查报告
1. 记录发现的漏洞及其严重程度。
2. 详细描述修复措施。

## 4. 执行建议
- 绝不妥协：对任何涉及用户隐私或系统安全的隐患实行“零容忍”。
- 记录坑点：将发现的安全易错点记录至 `AI_DEVELOPMENT_DOC.md`。
