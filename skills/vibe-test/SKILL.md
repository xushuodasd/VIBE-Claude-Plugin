# 测试工作流

## 1. 文档目的
规范测试代码的编写与执行流程，确保交付功能稳定可靠，杜绝未经验证的代码合入。

## 2. 工作流结构
- **前置步骤**：确认测试框架配置（如Jest, Vitest, PyTest等），读取待测代码。
- **执行步骤**：编写测试用例、执行测试、分析测试报告。
- **执行要求**：必须符合RED-GREEN-REFACTOR（红-绿-重构）原则。

## 3. 详细流程
### 步骤一：分析待测范围
1. 阅读相关需求文档（PRD）和接口文档。
2. 识别核心业务逻辑、边界条件及异常处理分支。

### 步骤二：编写测试用例
1. 编写失败的测试用例（RED），确保测试用例能真实反映需求。
2. 运行测试，确认测试如预期般失败。

### 步骤三：开发/修正业务代码
1. 编写最少量的业务代码，使测试通过（GREEN）。
2. 执行所有测试，确保没有破坏现有功能。

### 步骤四：重构与优化
1. 在测试通过的前提下，重构代码以满足工业级代码规范（模块化、接口化）。
2. 优化性能和可读性，重新运行测试确保依然通过（REFACTOR）。

## 4. 输出成果
- 通过的测试用例文件（通常在 `__tests__` 或 `tests` 目录下）。
- 测试覆盖率报告。
