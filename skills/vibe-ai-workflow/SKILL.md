# AI工作流文档规范

## 1. 文档目的
规范AI工作流的编写标准，确保AI能按用户要求生成并执行工作流。

## 2. 工作流结构

一个简洁高效的AI工作流文档应包含：

- **基本信息**：名称、目标
- **前置步骤**：产物检查、依赖项确认
- **执行步骤**：详细的执行流程和顺序
- **执行建议**：专业建议和注意事项
- **成功标准**：任务完成的判定条件
- **失败处理**：异常情况的应对措施
- **输出成果**：明确的交付物和保存位置

## 3. 执行要求

- 严格按照步骤执行
- 与用户保持深度沟通
- 记录关键信息和结果
- 遇到异常时按失败处理机制执行
- 确保输出成果符合用户预期

## 4. 文档管理

- 执行后根据实际情况更新文档
- 进行版本管理，确保使用最新版本

# 人物体系工作流

## 工作流程

### 前置步骤：产物检查
1. **检查创作规划文档**：
   - 确认`../小说基本信息文件夹/创作规划.md`文件是否存在
   - 如果不存在，提示用户先执行创作规划工作流

2. **检查世界观设定文件**：
   - 确认`../小说基本信息文件夹/世界观/`目录下是否存在世界观设定文件
   - 如果不存在，提示用户先执行世界观创作工作流

3. **读取相关文档**：
   - 读取创作规划文档，了解核心定位和目标读者
   - 读取世界观设定文件，了解世界背景和规则

### 第一步：与用户沟通确定前期主要角色
1. **角色范围界定**：
   - 与用户沟通："作为网文作者，您希望在前期重点塑造哪些核心角色？通常包括主角、1-2个重要配角（如女主/男主、男二/女二）和1个主要反派"
   - 明确前期主要角色清单，确保覆盖故事初期的核心人物

2. **创作目标确认**：
   - 与用户沟通："您希望这些角色具有什么样的特质和魅力？是热血逆袭、智计无双，还是情感细腻？"
   - 了解用户的创作偏好和角色定位

3. **优先级排序**：
   - 与用户共同确定角色创作的优先级
   - 建议顺序：主角 → 重要配角 → 主要反派
   - 确保每个角色都得到充分的创作时间和精力

### 第二步：与用户沟通创作主角
1. **基础信息确认**：
   - 与用户沟通："您希望主角的姓名、年龄、性别、身份和职业是什么？外貌形象有什么具体要求？"
   - 记录用户的具体要求，作为创作基础

2. **四维锚定沟通**：
   - 与用户沟通："您希望主角的核心标签是什么？外在目标和内在动机分别是什么？"
   - 与用户沟通："主角面临的核心障碍是什么？您希望他/她有怎样的成长弧光？"
   - 确保四维锚定符合用户的创作意图

3. **立体性挖掘引导**：
   - 与用户沟通："主角的性格特质、优点缺点是什么？有什么特殊癖好或口头禅？"
   - 与用户沟通："主角的能力特长、背景经历、情感世界如何设计？"
   - 与用户沟通："主角的生活细节和语言风格有什么特点？"
   - 引导用户深入思考，提供专业建议

4. **真实性确认**：
   - 与用户沟通："您希望主角有哪些独特的记忆和经历？"
   - 与用户沟通："主角有什么矛盾和挣扎？如何让他/她更真实可信？"
   - 确保人物真实立体，符合用户期望

5. **文档化**：
   - 根据与用户的沟通结果，使用[主角模板](../人物信息/人物模板/主角模板.md)填写详细信息
   - 将主角设定文档保存至`../人物信息/人物文档/`目录
   - 与用户确认文档内容，确保符合其创作意图

### 第三步：与用户沟通创作重要配角
1. **基础信息确认**：
   - 与用户沟通："您希望这个配角的姓名、年龄、性别、身份和职业是什么？外貌形象有什么具体要求？"
   - 记录用户的具体要求，作为创作基础

2. **独立设定沟通**：
   - 与用户沟通："这个配角的核心欲望、核心立场是什么？"
   - 与用户沟通："您希望这个配角有怎样的独立故事线和成长弧光？"
   - 确保配角有独立的人格和发展路径

3. **立体性挖掘引导**：
   - 与用户沟通："这个配角的性格特质、优点缺点是什么？有什么特殊癖好或口头禅？"
   - 与用户沟通："配角的能力特长、背景经历、情感世界如何设计？"
   - 与用户沟通："配角的生活细节和语言风格有什么特点？"
   - 引导用户深入思考，提供专业建议

4. **与主角关系确认**：
   - 与用户沟通："这个配角与主角的相识契机是什么？"
   - 与用户沟通："他们之间有哪些关键互动？配角对主角有什么影响？"
   - 与用户沟通："他们的关系会如何动态变化？"
   - 确保关系设计合理自然

5. **文档化**：
   - 根据与用户的沟通结果，使用[配角模板](../人物信息/人物模板/配角模板.md)填写详细信息
   - 将配角设定文档保存至`../人物信息/人物文档/`目录
   - 与用户确认文档内容，确保符合其创作意图

### 第四步：与用户沟通构建人物关系图谱
1. **关系网梳理**：
   - 在完成所有前期主要人物的独立创作后，与用户沟通人物关系
   - 与用户沟通："您希望这些角色之间有怎样的羁绊、恩怨和立场变化？"
   - 明确人物间的关系网络

2. **关系动态规划**：
   - 与用户沟通："您希望这些角色的关系如何动态发展？"
   - 规划人物关系的变化轨迹
   - 确保关系变化符合人物性格和剧情逻辑

3. **人设一致性检查**：
   - 与用户共同检查人物在关系中的行为是否符合其设定
   - 确保所有人物的互动自然合理

4. **文档化**：
   - 根据与用户的沟通结果，创建人物关系图谱文件
   - 将关系图谱文件保存至`../人物信息/人物文档/`目录
   - 与用户确认图谱内容，确保符合其创作意图

### 第五步：后期角色创作说明
1. **后期角色范围**：
   - 明确本工作流仅负责前期主要角色的创作
   - 后期出现的其他角色将通过专门的后期角色创作工作流进行处理

2. **前期与后期的衔接**：
   - 确保前期主要角色的设定为后期角色的引入预留空间
   - 保持人物体系的一致性和扩展性

## 执行建议
1. **深度沟通**：作为网文大家，与用户进行深入对话，了解其创作意图和角色期望
2. **专业引导**：基于丰富的创作经验，引导用户思考角色的深度和细节，提供专业建议
3. **专注创作**：每次只专注于一个人物的创作，确保充分挖掘其深度和细节
4. **循序渐进**：按照优先级顺序创作人物，先完成一个再开始下一个
5. **细节挖掘**：引导用户关注人物的生活细节、语言风格和情感变化，让人物真实可信
6. **逻辑自洽**：确保人物行为和决策符合其性格和背景，避免前后矛盾
7. **独立人格**：每个人物都应有自己的思想和判断，不是情节的工具
8. **关系自然**：人物间的互动应真实自然，符合关系定位和各自的性格
9. **动态发展**：人物应随剧情发展而成长变化，展现真实的成长弧光
10. **避免套路**：突破传统人物设定套路，创造独特有记忆点的角色
11. **前期聚焦**：专注于前期主要角色的创作，为故事打好基础
12. **预留空间**：确保前期角色设定为后期角色的引入预留空间

## 输出成果
1. 完整的主角设定文档（包含四维锚定和深度细节，与用户沟通确认） - 保存至`../人物信息/人物文档/`目录
2. 重要配角设定文档（每个都有独立故事线和成长弧光，与用户沟通确认） - 保存至`../人物信息/人物文档/`目录
3. 前期主要人物关系图谱（展示所有前期主要人物间的关系网络，与用户沟通确认） - 保存至`../人物信息/人物文档/`目录
4. 前期主要人物成长轨迹规划（每个人物的独立成长路径，与用户沟通确认） - 保存至`../人物信息/人物文档/`目录

记住，好的人物是故事的灵魂。一个立体、有弧光的人物能让读者产生强烈的情感共鸣，为作品增添持久的魅力。
