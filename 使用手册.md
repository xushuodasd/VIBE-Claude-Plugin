# VIBE-Claude-Plugin 快速使用手册

欢迎使用 VIBE-Claude-Plugin。这是一套为 Claude Code 打造的“24小时全自动开发引擎”，通过模拟“研发总监+员工”的多智能体架构，实现企业级的无人值守编程。

---

## 1. 快速安装

由于 Claude Code 的安全机制，你需要通过 Git 将插件克隆到 Claude Code 的官方插件目录中。

**Windows (PowerShell) 一键安装:**
```powershell
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.claude\plugins" | Out-Null
git clone https://github.com/xushuodasd/VIBE-Claude-Plugin.git "$env:USERPROFILE\.claude\plugins\vibe-claude-plugin"
```

**macOS / Linux 一键安装:**
```bash
git clone https://github.com/xushuodasd/VIBE-Claude-Plugin.git ~/.claude/plugins/vibe-claude-plugin
```

安装完成后，**彻底重启 Claude Code**（按 Ctrl+C 退出后重新输入 `claude` 启动），插件中的所有技能将自动加载。

---

## 2. 核心功能：如何开启 24 小时全自动开发？

为了让 AI 能够不间断干活而**不会停下来向你索要 `y/N` 权限**，请务必按照以下步骤操作：

### 步骤 A：赋予终端执行权限（关键）
在 Claude Code 刚启动时，建议使用 `--trust` 参数，或者在它第一次提示要不要执行终端命令时，选择 **`Yes, don't ask again for this session`**。

### 步骤 B：使用“魔法指令”触发引擎
在聊天框中，使用 `开启全自动开发` 作为前缀来描述你的需求。

**标准话术示例：**
> "开启全自动开发：我要做一个前端待办事项网站。要求深色工业风，按钮有物理反馈感。后端用 Node.js 存 SQLite。"

## 3. 全自动开发期间会发生什么？

引擎启动后，分为两个明确的阶段：

### 阶段一：需求对齐（它会提问）
在这个阶段，AI 扮演**产品经理与架构师**。
- 它会根据你的话术生成 PRD（需求文档）、API 接口文档和架构设计。
- **注意**：此时它可能会向你提问以确认细节（例如：“主题色你想偏蓝还是偏绿？”）。
- 当一切敲定后，它会生成一份 `tasks.md` 任务清单，并问你：“是否可以开始全自动开发？”
- 你回复：**“可以，开始吧”**。

### 阶段二：无人值守开发（它将彻底闭嘴）
一旦你同意开始，AI 将化身为**研发总监和底层打工人**的结合体。
- **彻底静默**：它不会再问你任何问题。遇到任何代码报错，它会自己寻找最优解修复。
- **工作流循环**：它会自动切分支 -> 查 API 文档 -> 写一个最小模块代码 -> 跑测试 -> 查安全漏洞 -> 加 UI 动效 -> `git commit` 提交 -> 在 `tasks.md` 打勾 -> 做下一个任务。
- 你可以去喝杯咖啡，直到它最终把交付报告打印在屏幕上。

## 4. 常见问题 (FAQ)

**Q: 如果中途我想打断它或者修改需求怎么办？**
A: 直接在终端按 `Ctrl+C` 强行打断它。跟它说你的新想法，它会重新更新 `tasks.md`，然后你再让它继续。

**Q: 跑了很久以后，感觉它脑子变慢了或者开始胡言乱语怎么办？**
A: 这是大模型的“上下文溢出”现象。直接输入 `/clear` 清空聊天记录。然后对它说：**“读取 tasks.md，继续全自动开发”**。它会瞬间满血复活，找回研发总监的状态继续干活。

**Q: 它可以接手我现有的半成品项目吗？**
A: 可以。只要输入 `开启全自动开发：分析当前项目，并帮我增加一个XXX功能`，它会自动去读你现在的代码并生成相应的计划。
